## Цель работы
Изучение концепций итераторов, генераторов и сопрограмм в Python. Реализация двух способов создания итераторов для фильтрации чисел Фибоначчи и изучение принципов работы сопрограмм.
## Теоретическая часть
1. Итератор — объект с методами `__iter__()` и `__next__()`.
Первый возвращает сам итератор, второй — следующий элемент или исключение `StopIteration`.
2. Итерируемый объект возвращает итератор через `iter()`. Может реализовываться через `__iter__()` или `__getitem__()` для доступа по индексу.
3. Генератор — функция с `yield`, автоматически создающая итератор. Состояние сохраняется между вызовами.
4. Сопрограмма — генератор, принимающий данные через `send()`. Обеспечивает двустороннюю связь с вызывающим кодом.
## Выполнение работы
### Задание 1.1
Был реализован классический итератор, последовательно обрабатывающий коллекцию данных.
Элементы проверяются и возвращаются по мере запроса, что позволяет эффективно работать с большими объемами данных.

Класс `FibonacciLst` фильтрует числа Фибоначчи из последовательности. 
Хранит позицию в атрибуте `idx`, возвращает только числа Фибоначчи. 
Преимущество — хранение состояния. Недостаток — однократное использование.
### Задание 1.2
Реализован альтернативный вариант итератора, использующий другой подход к обработке данных. 
В отличие от первого варианта, этот итератор предварительно фильтрует всю коллекцию при создании, 
что обеспечивает быстрый доступ к элементам по индексу.

Класс `FibonacciLstSimple` реализует `__getitem__()`, возвращая элементы по индексу. 
Не хранит состояние, позволяет многократную итерацию и прямой доступ к элементам. 
Менее эффективен — каждый вызов сканирует последовательность заново.
### Задание 2
Реализована специализированная сопрограмма, демонстрирующая принцип кооперативной многозадачности. 
Данная реализация использует механизм `yield` для организации двустороннего обмена данными между вызывающей и вызываемой сторонами.

Функция `fib_gen()` принимает количество чисел через `yield` и возвращает список Фибоначчи. 
Использует метод `send()` для двустороннего взаимодействия. 
Декоратор `start()` автоматически инициализирует сопрограмму через `next()`.
